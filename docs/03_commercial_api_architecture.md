# AI Video Material Studio: Commercial API Hybrid Architecture
Version: 2.0.0
Date: 2026-04-05

## 1. 概要
本仕様書は、既存のローカルGPU完全依存型オープンソース・パイプライン（Wan 2.1 / FLUX 等使用）を、**Kling AI** や **Hedra** などの商用トップレベルのAPIと連携させる「APIハイブリッド・パイプライン」へ移行するためのアーキテクチャ再構築プランを定義します。

**目標**: 
競合（Sora, Veo 3）やトップクリエイター（麻哥等）の手作業によるワークフローを凌駕する映像品質（1080p相当の解像度、完全な物理整合性、および高精度なリップシンク）を、完全自動化されたシステム上で実現すること。また、生成時間を数十分から「数分以内」へ短縮し、プラットフォームのSaaSとしての完全な商用運用を可能にします。

---

## 2. パイプライン構成（V2 新アーキテクチャ）

本アーキテクチャは中国・英語圏のトップAI動画クリエイターが手動で行っている「静止画固定（SDXL/FLUX）+ I2V（Kling/Minimax）+ 編集ソフト合成」のアプローチをベースとし、`main.py` のモジュールを以下のようにAPIコールへ刷新する形で実装します。

### ステージ 1: キャラクター固定・ベース画像生成
*   **エンジン**: **Fal.ai (FLUX.1 Pro / Dev API)** 
*   **手法**: 
    *   IP-Adapter、あるいはLoRAを用いて、指定されたキャラクター（女性モデル、男性モデル等の自社定義アバター）の顔立ちを厳密に固定した静止画（720x1280等の縦型比率）を高速生成。
    *   ユーザー指示の背景プロンプト（「オフィス」「レストラン」「リビング」など）を反映し、複数シーンのベース画像を確実に一貫性を持って生成します。
*   **利点**: 自社A4500のVRAM限界による妥協を廃し、Soraと同等の鮮明なグラフィックの基盤が確立します。

### ステージ 2: Image-to-Video（動画化と空間物理演算）
*   **エンジン**: **Kling AI API** または **Runway Gen-3 Alpha API**
*   **手法**:
    *   ステージ 1 で生成されたキャラクター画像URLと、動きの指定プロンプト（`slow camera pan`, `subtle breathing, natural smile`等）をKling APIに渡し、3〜5秒の商用クオリティクリップを生成。
    *   Wan 2.1 14Bモデルのローカル生成ではどうしても消しきれなかった「肌ののっぺり感」や「指の溶け」「背景の歪み」を、商用APIの最高精度で解決します。
*   **利点**: 長時間（40分以上）のローカル計算が、APIコールのみ（1-2分）へと圧倒的に短縮されます。

### ステージ 3: 音声合成 (TTS)
*   **エンジン**: **Style-Bert-VITS2 (既存のローカルサーバー)** または **ElevenLabs API**
*   **手法**: 台本解析に基づく日本語ネイティブな高品質ナレーション音声を担保します。これは従来通り稼働が軽いため、自社サーバー・ローカル環境のTTSエンジンを継続利用するか、より感情豊かなElevenLabs APIへの移行を選択します。

### ステージ 4: 高精度リップシンク（口パク合成）
*   **エンジン**: **Hedra API** または **SyncLabs API**
*   **手法**: 
    *   ステージ 2のKlingで生成された動画と、ステージ 3の音声をHedraに入力。動画内のアバターの口を、音声の抑揚に合わせて100%の精度で一致させます。
*   **利点**: トップクリエイターでさえ妥協しがちな「なんとなく口が動いているだけ」の違和感を排除し、企業PVに使える「本物のアナウンサー」のアウトプットを提供します。

### ステージ 5: 編集・合成・テロップ (自動化)
*   **エンジン**: **Whisper API + FFmpeg (ローカル実行)**
*   **手法**: 
    *   LipSyncが完了した複数シーンの動画を結合。
    *   BGMと合成し、AI自動音声認識(Whisper)により全フレームへの**自動テロップ（字幕等）の焼き付け**を行います。（CapCutの手作業をプログラムで完全自動化）。
    *   左上に指定された「AI生成」ウォーターマーク・ロゴを挿入します。

## 3. 実装・改修フェーズ

| フェーズ | 対象ステージ | タスク内容 |
| :--- | :--- | :--- |
| **Phase 1** | Stage 4 (I2V動画化) | `main.py`のWan2.1(ComfyUI)呼び出しを非アクティブ化し、Kling API（またはその他選定API）用リクエストモジュールを新規実装。APIキー認証とWebhook（またはポーリング）による結果取得フローの構築。 |
| **Phase 2** | Stage 1, 2 (画像生成) | BlenderとローカルFLUXのフローを刷新。Fal.ai等へのAPIリクエストに置き換え、FaceID/IP-Adapterによるキャラクター完全固定化のプロンプトエンジニアリングをコード内に組み込む。 |
| **Phase 3** | Stage 4 (リップシンク) | 現在の `liveportrait_lipsync` フローを Hedra API に置換。 |
| **Phase 4** | Stage 5 (編集・合成) | FFmpegスクリプトの強化。動的テロップ生成とBGMオーバーレイ機能の実装。 |

## 4. 費用対効果とスケーラビリティ
ローカルのVRAM制限（A4500 20GB等）によるボトルネックから完全に脱却することで、ユーザー数が増大してもAPIを介した水平スケールが無制限に可能となります。
GPUサーバー側の負担は「音声合成と最終のFFmpeg結合」など超軽量タスクのみに限定されるため、安価なサーバー（または既存の自社プラットフォーム環境を流用可能）で何十件ものジョブを並列処理する圧倒的コスパを実現します。
